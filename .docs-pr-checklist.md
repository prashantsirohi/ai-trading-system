# Docs PR Checklist

- Did I change actual runtime behavior?
- Did I update the canonical docs in `docs/` in the same change?
- Did I avoid creating a second doc that explains the same thing differently?
- Did I mark legacy, compatibility, or experimental paths clearly?
- Did I verify the documented commands and entrypoints still exist?
- Did I update env vars, flags, artifacts, endpoints, or UI references if they changed?
- If the data source-of-record changed, did I update pipeline, installation, runbook, and commands docs?
- If a new endpoint was added, did I update `docs/interfaces/api.md`?
- If a new artifact was added, did I update `docs/reference/artifacts.md`?
